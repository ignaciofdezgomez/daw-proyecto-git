Artículo de GIT

GIT

Git es un sistema de control de versiones open source que ayuda a los equipos de desarrollo de software a trabajar de una manera distribuida y flexible sus proyectos. Git fue lanzado el año 2005 por Linus Torvalds que en ese entonces estaba buscando una mejor herramienta para gestionar parches y colaboraciones en el código del kernel Linux.

Sistema de control de versiones

Los sistemas de control de versiones son sistemas capaces de rastrear y gestionar los cambios en el código fuente. Estos sistemas brindan a los equipos múltiples herramientas para que puedan registrar los cambios a través del tiempo. Los software de control de versiones realizan seguimiento de todas las modificaciones en el código en un tipo especial de bases de datos. Si los desarrolladores cometen un error, es posible volver hacia atrás en el tiempo para comparar versiones. Los sistemas de control de versiones no imponen una forma determinada para trabajar pero, si brindan una serie de directrices y comandos para realizar todo tipo de flujo de trabajo. Podemos clasificar los sistemas de control de versiones dependiendo de su arquitectura: Locales, centralizadas y distribuidas. .

  1. Locales: Los cambios son guardados localmente y no se comparten con nadie. Esta arquitectura es la antecesora de las centralizadas y distribuidas.
  2. Centralizada: Los cambios son guardados localmente y no se comparten con nadie. Esta arquitectura es la antecesora de las centralizadas y distribuidas.
  3. Distribuidas: En esta arquitectura cada usuario tiene su propio repositorio y los distintos repositorios pueden intercambiar y mezclar revisiones entre ellos. Entre ellos se encuentran Git y Mercurial.
